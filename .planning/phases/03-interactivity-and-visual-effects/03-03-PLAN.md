   - Target the hero heading/name element (e.g., "I'm Khai Phan." text)
   - Add class `e6-hero-name` to the hero name element for targeting
   - In useGSAP: `const heroSplit = new SplitText(".e6-hero-name", { type: "chars" });`

   - This replaces or enhances any existing hero name animation. Ensure it fits within the existing hero timeline if one exists.

   **Section heading reveals (word split):**
   - Target all `.e6-split-heading` elements
   - For each heading: `const split = new SplitText(heading, { type: "words" });`
   - Animate: `gsap.fromTo(split.words, { y: 40, opacity: 0 }, { y: 0, opacity: 1, duration: 0.6, stagger: 0.08, ease: "power3.out", scrollTrigger: { trigger: heading, start: "top 85%" } });`
   - IMPORTANT: SplitText must run client-side only (inside useGSAP). The heading renders normally on server; SplitText wraps words on client. The opacity:0 fromTo start means the brief unsplit state is invisible.

   - Review existing gsap.fromTo() + ScrollTrigger animations on sections (about, experience, skills, projects, contact)

   - If any sections use gsap.from() instead of gsap.fromTo(), convert to fromTo() (established anti-pattern from Phase 2)
   - Ensure the double-rAF + ScrollTrigger.refresh(true) pattern is still applied at the end

   **SplitText cleanup:** SplitText instances should be reverted on component unmount. useGSAP handles GSAP cleanup, but also call `split.revert()` in cleanup if needed for DOM restoration.
