- Bright flat backgrounds, no gradients
- Elements with slight CSS rotation (`rotate-1`, `-rotate-2`)

**Exploration 5: Video Game-Inspired** (`Exploration5.tsx`)
Replace the placeholder with a complete design exploration showing Hero + Skills section.

Visual identity:
- Typography: Techy, futuristic sans-serif or pixel-adjacent. Think Orbitron, Press Start 2P for accents, or Rajdhani. Monospace for "terminal" elements. Mix a display font for headings with a clean sans for body.
- Color palette: Dark background (#0a0a0a or deep navy #0d1117), neon accents -- cyan (#00e5ff), green (#39ff14), magenta (#ff00ff), amber (#ffab00). Colors should glow against the dark background. Think HUD overlay on dark screen.
- Layout: HUD/game UI aesthetic. Corners of the viewport have decorative UI elements (coordinate readouts, status indicators, thin border frames). Content organized in "panels" with thin neon borders. Hero text appears as if being typed or loaded. Skills section displayed as a "stats" screen or "inventory" with progress bars or level indicators.
- Animation: Digital/glitchy GSAP -- text typing effect (characters appearing one by one via stagger), elements that flicker briefly on entrance, neon glow pulsing (CSS animation or GSAP repeat). Progress bars that fill from 0 to value. Status elements that "boot up" sequentially.
- Mood: You just entered the developer's command center. Game UI meets portfolio. Bruno Simon's playfulness at CSS-only fidelity. Interactive, techy, memorable.

CSS techniques required:
- Neon glow: `text-shadow: 0 0 10px #00e5ff, 0 0 40px #00e5ff` or `box-shadow` glow
- Thin border frames: `border border-cyan-500/30` with corner accents
- Dark background with grid pattern or scanlines (CSS background-image repeating-linear-gradient)
- Monospace text for "data" elements
- CSS animations for pulsing glow effects

Content for Hero: "KHAI_PHAN" in monospace/tech font with a typing animation, "// Software Engineer" as a code comment subtitle, decorative HUD elements (fake coordinates like "LAT 37.7749 / LNG -122.4194", a "STATUS: ONLINE" indicator).
Content for Skills: Skill categories (Frontend, Backend, Tools) displayed as game-style stats with neon progress bars filling to different levels, tech names in monospace.

HUD decoration elements (positioned absolute in corners):
- Top-left: thin L-shaped corner bracket
- Top-right: "coordinates" or timestamp text
- Bottom-left: "SYSTEM STATUS" type readout
- Bottom-right: navigation indicator
These should be subtle, decorative, and add atmosphere without cluttering content.

Component must:
- Be "use client" with useRef and useGSAP (scope: containerRef)
- Use Tailwind utilities + inline styles for neon effects (text-shadow not available in Tailwind by default)
- Be self-contained with all visual identity defined within
- Keep it CSS + GSAP only -- NO canvas, NO WebGL, NO Three.js per project scope
- Render at full viewport height for Hero, natural height for Skills
